==== Entidades Principales

* *RESERVATION*: Entidad central que representa una reserva de cancha
* *RESERVATION_TYPE*: Tipos de reserva disponibles
* *OPERATOR*: Operadores del sistema
* *CLIENT*: Clientes que realizan reservas
* *CLIENT_TYPE*: Tipos de cliente
* *COURT*: Canchas disponibles para reservar (con campos `cost` y `price`)
* *COURT_TYPE*: Tipos de cancha
* *PAYMENT*: Pagos asociados a reservas. Puede existir sin reservación asignada (pagos huérfanos). El webservice registra pagos recibidos sin conocer la reservación, y luego cuando el usuario está pagando, si encuentra el pago huérfano, lo alinea con la reservación.
* *DISCOUNT*: Descuentos aplicados a reservas
* *TARIFF_ADJUSTMENT*: Ajustes de tarifas aplicables según tipo de cliente, tipo de reserva y rangos de fechas/horas (RFC 5545)
* *USER*: Usuarios del sistema (interfaz con Firebase Auth)

=== Código Limpio

* SOLID principles

* KISS (Keep It Simple, Stupid)
* YAGNI (You Aren't Gonna Need It)

=== Conventional Commits

Formato estándar para mensajes de commit:

[source]
----
<type>(<scope>): <subject>

<body>

<footer>
----

Tipos comunes:
* `feat`: Nueva feature
* `fix`: Corrección de bug
* `docs`: Documentación
* `style`: Formato
* `refactor`: Refactorización
* `test`: Tests
* `chore`: Tareas de mantenimiento

=== Semantic Versioning

Versionado semántico: `MAJOR.MINOR.PATCH`

* *MAJOR*: Cambios incompatibles
* *MINOR*: Nuevas features compatibles
* *PATCH*: Correcciones compatibles
